$ lex-scripta --guarantees

原始笔记会丢什么，Lex Scripta 会保留什么。

自然语言不变，作用域固定。

git diff --word-diff notes.md guarantees.md
会丢掉的东西 failure-modes.md
  1. @@@ informal note failure modes
  2. -日期看起来像已经确定。
  3. -例外会脱离它真正修改的规则。
  4. -负责人会埋进大段 prose 里。
  5. -复核时必须重读整段文字。
会留下的东西 guarantees.md
  1. @@@ Lex Scripta guarantees
  2. +未定日期会保持未定。
  3. +例外仍挂在正确的父规则下面。
  4. +负责人会变成稳定的条款。
  5. +复核可以逐行进行。

$ cat structure-rules.md skeleton.md

Lex Scripta 是可读的规则树。

父行立规则，缩进绑定范围。

cat structure-rules.md skeleton.md
结构规则 structure-rules.md
  1. @@@ Lex Scripta document shape
  2. 一行只放一条父规则。
  3. 缩进后的子行继承父规则范围。
  4. 子行可以表达条件或例外。
  5. 负责人和未定事项也保持为显式条款。
  6. 空行用来分隔独立规则块。
文档骨架 skeleton.md
  1. +发布策略管理 Beta 注册规则。
  2. + 发布时间目标是下个月。
  3. + 具体日期仍未确定。
  5. +注册先走邮箱优先规则。
  6. + 初期只收邮箱。
  7. + 手机验证延后处理。
  8. + 企业账号可能需要公司邮箱验证。
  10. +设计由 Minji 负责。
  11. +发布负责人仍未确定。
